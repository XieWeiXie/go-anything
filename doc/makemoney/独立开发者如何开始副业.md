大家好，我是壹树，主业是一枚产品经理，副业目前专注于  小众市场，虽然说这两年才搞浏览器插件副业，但我的副业从 08 年就已经开始了。

一、寻找方向，小众开始

很多加我微信的朋友，技术都不错的，大部分都比我好，但最主要的问题是不知道方向，或者即使知道方向了，也没有办法坚持下去。

有些人认为浏览器插件都是免费的，当然不是，看你通过什么方式变现而已，免费的也能赚钱。

若想寻找方向，开始自己的副业，我推荐下面以些小众市场。

Microsoft AppSource	
Shopify App	
G Suite	
Firefox  add-ons	
Opera add-ons	
Adobe Exchange	
Envato	

我为什么推荐这些市场呢？因为大公司基本上忽略这些市场，不会出现例如：腾讯也做了你怎么办，自己一个人也可以做一个小插件满足用户的特定需求，解决用户的特定问题。

小众很多时候只是对比的的结果，例如谷歌浏览器的市场份额占是市场的 70% ，每月的活跃用户有 10 亿，这些数据看起来都不小的，我们需要做的只是满足这里面的极小部分人的需求。

例如微软的 office 系列的插件，整个生态很大的，只是我们盲目地认为小了而已。

又例如 Shopify 这家公司，你可以理解为海外的有赞，就是为几百万的店家提供开店工具，里面的插件，很多都是个人或者小团队做的，这种属于天然用户聚焦的地方。市场已经帮你把店家用户聚起来了，就看你如何去分这块蛋糕了。

我再推一个目前我接触的最牛逼的插件例子，单单围绕 Gmail 就做了将近 20 款插件。为什么会这样做，因为 Gmail 用户有 15 个亿。你哪怕能服务 0.01%  的用户都不得了。

二、摆脱技术依赖，锻炼自己解决问题的思维

在开始副业之前，我是建议尽量摆脱对技术的依赖，我们上班很多时候只是贡献代码。

但如果你面对一个市场，就相当于直面用户，你需要的是帮用户解决问题，代码只是一种方式，插件也只是一种方式。

还有很多帮用户解决问题的方式，你只需要选其中一种方式给到用户，用户就有可能付费了。赚钱这一步是后置的，帮用户解决问题才是优先。

例如，我有一个简单的插件，大概花了 2 小时，用的是 github 上面的库，封装一下就上线了，就是让用户看视频评论的时候可以选择表情，因为 Youtube 上评论框是没表情选的。就是这么简单的点。



还有另外一个例子，之前我的公众号分享一篇非常完整插件文章，我只是通过插件在 Youtube 上加了一个下载 MP3 的按钮，就能解决用户的问题，用户就愿意付费了。

Chrome 插件过山车之旅


你看，上面的插件需要用到的技术都不复杂，我只是以一种合适的方式呈现到用户面前，刚好解决了用户的问题，回报就是伴随着问题的解决而出现在我面前的。

所以，先把问题解决了，然后再理直气壮地向用户收钱。

三、能做的只有 2 种产品

有一种说法，互联网只有 2 种产品，一种帮你花时间的，一种帮你省时间的。娱乐，游戏等帮你快速花去时间，对于省时间的就更多了，随便从身边拿起一样东西，很大概率是省时间，这种省时间的核心产品就是帮用户解决某个特定问题。

所谓的产品思维我的理解就是解决问题的思维，而这个思维第一步就是先培养自己发现问题的能力。一般是通过两种途径发现问题。

第一，从自身的问题出发，自己生活中，工作中会遇到各种问题，这些问题都是我们产品的起点，有些问题可能我们很轻松解决，有些会一直让自己不爽，这些不爽就是痛点。

但我说的不是首先围绕这些不爽去找答案，而是先罗列自己能轻松搞定的问题，这些问题我们是能快速找出答案，因为我们具备特定的能力。但其他人没有。

第二，从他人身上发现，例如身边的朋友，网络社群上的人的讨论，让自己的好奇心活动起来，看到问题后，多思考若这个问题是出现在我们自己身上，将会以怎样的方式去解决，罗列出步骤 1、2、3， 4，到这里你会发现又回到第一点了。这就是常说的：用户思维，代入用户的角色。

四、如何开始一个产品

那么如何开始我们的一个产品呢？这里我分享一下我的产品挖掘流程，目前我基本按这样的思路去搞我的插件。

平台 → 用户 → 问题 → 需求 →  竞品 →  数据 → 决策 → 产品

我简单讲解一下这个流程，选定一个大平台（为了细分用户），专注服务于这个平台的用户，这过程需要体验平台的服务，尝试找出用户的问题，为问题找出答案，这个答案就是用户需求，然后再看看是否有竞品，竞品是如何帮用户解决问题，对比一些可获得的数据，最后拍脑门选一个答案，形成自己的产品。

产品挖掘，这是一个比较大的课题，这里就不展开了，后续我的课程会有详细的讲解。这个流程就是目前我的思维方式。

对于小产品，小插件， MVP 理论是很受用的，也就是最小可用产品，一个功能就满足用户，浏览器插件的官方宗旨也是这样。

MVP 是一个设计过程，很好的表达了单一功能的设计理念。

五、好奇心，必须具备

在副业或做产品的过程中，若说是压力为主，其实不是的，压力有时候会扭曲的，会让我们变得无助。

如我们的产品为什么没有用户？用户有了，但为什么不付费？用户付费了，但为什么续费率那么低？这些都会对我们造成压力。适当的压力还好，如果过度了，容易让自己的心态扭曲。

所以我建议在过程中，把好奇心放到第一位，为什么呢？好奇心是我们积极探究新奇事物的一种冲动，强烈的好奇心会增强人们对外界信息的敏感度，容易让我们发现问题，推动我们去找答案。

六、好的心态，好的开始

你开始做第一个产品的时候，用户来得很慢，可能因为对用户理解不深，用户不一定买账，这都是一个过程，时长时短，通过持续的收集用户反馈，肯定能让一部分用户满意的，钱自然就来了。

更多的时候不是我们做得不对，只是我们没法持续做一件我们认为对的事情。可能你走多一步就有第一付费用户了，但你放弃了，你说可惜不可惜。副业必须要耐住寂寞，有一种坚持的信念才行。所以一定要保持好的心态。
